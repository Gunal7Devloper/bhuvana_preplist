1. What is DevOps?

DevOps is a set of practices, tools, and a cultural philosophy that aims to unify and automate the work of software development (Dev) and IT operations (Ops). The goal is to shorten the development lifecycle, deliver high-quality software continuously, and improve collaboration between development and operations teams.

It focuses on:

  • Continuous Integration and Continuous Delivery (CI/CD)
  • Automation of infrastructure and workflows
  • Monitoring and performance feedback
  • Collaboration and shared responsibilities

DevOps promotes using tools like Git, Jenkins, Docker, Kubernetes, Ansible, and cloud platforms like AWS or Azure to automate and streamline the software delivery process.

Bonus Tip: If asked for real-world benefits, you can add:
With DevOps, teams can release features faster, detect bugs earlier, and respond to customer needs more efficiently.

1. What is Nginx?

Nginx (pronounced "engine-x") is a high-performance, open-source web server software that is also widely used as a:

  • Reverse Proxy
  • Load Balancer
  • HTTP Cache
  • API Gateway

üîßWhat Can Nginx Do?

Use Case Description
Web Server Serves static content (HTML, CSS, JS, images) directly to users.
Reverse Proxy Forwards client requests to backend servers (e.g., Node.js, Flask, Django) and returns the response to the client.
Load Balancer Distributes incoming traffic across multiple servers to improve performance and reliability.
SSL Termination Handles HTTPS/SSL encryption/decryption to offload that work from backend apps.
Caching Caches static or dynamic content to reduce load on backend servers.

üîçWhy Is It Popular?

  • Handles high concurrency with low memory usage (event-driven, non-blocking architecture).
  • Faster than traditional servers like Apache in many use cases.
  • Used by many high-traffic websites like Netflix, Dropbox, and WordPress.com.

‚úÖ Step-by-Step: Deploy Angular with Nginx

1. Build the Angular app

In your Angular project root:

bash
ng build --configuration production

This creates a dist/your-app-name/ folder with the production-ready static files.

2. Install Nginx

On Ubuntu/Debian:

bash
sudo apt update
sudo apt install nginx

3. Copy build files to Nginx web root

bash
sudo cp -r dist/your-app-name/* /var/www/html/

Or to a custom directory like /var/www/my-angular-app/.

4. Configure Nginx

Edit the config:

bash
sudo nano /etc/nginx/sites-available/default

Replace with something like:

nginx
server {{ '{' }}
listen 80;
server_name yourdomain.com;

root /var/www/html;
index index.html;

location / {{ '{' }}
try_files $uri $uri/ /index.html;
{{ '}'}}
{{ '}'}}

try_files is important for Angular‚Äôs HTML5 routing (e.g., /dashboard, /profile).

5. Restart Nginx

bash
sudo systemctl restart nginx
